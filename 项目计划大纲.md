# 广东省高考志愿填报参考系统 — 项目计划大纲

> **目标用户**：广东省高考考生
> **核心价值**：根据高考分数与排名，结合历年数据，提供科学合理的志愿填报参考建议

---

## 一、项目概述

### 1.1 项目定位
面向广东省高考生的志愿填报辅助参考平台，帮助考生在"新高考"模式下（3+1+2），根据自身分数、排名、选科组合，智能推荐院校与专业，并按照"冲、稳、保"三级策略生成志愿填报方案。

### 1.2 广东省高考特点（需重点考虑）
- **模式**：新高考"3+1+2"（语数英 + 物理/历史 + 两门选考）
- **批次划分**：本科批、专科批（本科批合并，不再分一本二本）
- **分数线类型**：
  - 特殊类型招生控制线（相当于原一本线）
  - 本科线
  - 专科线
- **选科要求**：各院校专业对选考科目有明确要求（物理/历史/不限等）
- **排名的重要性**：广东考生数量大，排名比分数更具参考价值
- **志愿数量**：本科批可填 45 个院校专业组志愿

### 1.3 核心原则
- **排名优于分数**：每年试题难度不同，分数线波动大，但排名相对稳定。例如 2026 年数学难度较大（与 2022 年类似），同样的分数对应的排名会比"简单年份"更高，因此排名才是跨年份对比的"硬通货"
- **分数差值直观呈现**：在推荐结果中，用"比我高 15 分"这样的分数差值作为主要展示语言，排名差值作为辅助参考，让考生和家长一目了然
- **±50 分推荐区间**：高于 50 分的院校考不上（不浪费用户注意力），低于 50 分的院校浪费分数（不值得推荐），聚焦有效区间
- **年份难度对标**：通过一分一段表分析各年份的分数-排名分布曲线，识别出"难度相似年份"（如 2026≈2022），在预测当年分数线时，给难度相似年份的权重更高，从而提高预测准确度
- **年份权重用户可控**：默认等权平均，同时支持用户手动调整各年份权重或一键应用难度对标权重
- **冲稳保策略**：科学划分三个层次，合理分配志愿数量（冲 15-20 个、稳 15-20 个、保 5-10 个）
- **数据驱动**：基于近 4-5 年官方录取数据进行推算，难度年份加权
- **选科匹配**：严格校验考生选科与院校专业要求的匹配性
- **专业组透明化**：标注专业组包含的具体专业方向及冷热程度，揭穿"好学校差专业低分"的陷阱
- **省内外无差别推荐**：不限制院校地域，主动挖掘省外高性价比院校
- **专业方向深度展开**：根据考生兴趣方向，展示大类下的细分专业投档线

## 二、功能模块设计

### 2.1 用户输入模块（个人信息采集）
| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 高考总分 | 考生查询到的最终分数 | ✅ 必填 |
| 全省排名（位次） | 广东省教育考试院公布的排名 | ✅ 必填 |
| 首选科目 | 物理 / 历史 | ✅ 必填 |
| 再选科目（2门） | 化学/生物/政治/地理 中任选2门 | ✅ 必填 |
| 目标省份/地区偏好 | 省内/省外/无偏好/具体省份（如北京、上海、湖北等） | 可选 |
| 目标专业方向 | 工学/医学/经管/文史/师范/法学/艺术等大类 | 可选 |
| 目标细分专业 | 在选定大类下进一步选择（如工学→计算机类→软件工程） | 可选 |
| 院校层次偏好 | 985/211/双一流/公办本科/民办本科/专科 | 可选 |
| 学费承受范围 | 公办标准/可接受民办/中外合作办学 | 可选 |

**省份偏好设计说明**（⭐ 相比原"城市偏好"的改进）：
- 原设计为"目标城市偏好"，现改为"目标省份/地区偏好"
- 理由：在分数和院校层次面前，具体城市是次要因素；先圈定省份范围更实用
- 同时保留"不限"选项，不限制省外好学校的推荐
- 层级设计：全国（默认）→ 大区（华北/华东/华南等）→ 省份 → 城市（逐步细化）

**专业方向选择层级设计**（⭐ 新增）：
```
第一层（大类）：工学 / 理学 / 医学 / 经管 / 文史哲 / 法学 / 
               师范 / 农学 / 艺术 / 体育 / 军事
第二层（中类，以工学为例）：计算机类 / 电子信息类 / 机械类 / 
                            土木建筑类 / 材料类 / 电气类 / ...
第三层（细分，以计算机类为例）：计算机科学与技术 / 软件工程 / 
                                人工智能 / 大数据 / 网络空间安全 / ...
```
> 📌 考生可以只选择第一层大类，也可以深入到第三层；选择的层级越细，推荐结果越精准。不选择则不限制专业方向。

**扩展考虑**：
- 是否有加分项（少数民族、华侨等）
- 是否有体检限制（色盲色弱等影响专业选择）
- 是否为农村专项/地方专项计划考生

### 2.2 数据查询与浏览模块
- **院校库**：
  - 全国高校基本信息（所在地、层次、类型、是否985/211/双一流）
  - 各院校在广东的招生计划（分专业组、分专业）
  - 院校历年最低投档线 & 对应排名
  - 院校各专业组最低投档线 & 对应排名
- **专业库**：
  - 各专业详细介绍、培养目标、主要课程
  - 就业方向与前景分析
  - 各专业在广东的历年录取分数 & 排名
  - 专业对选科的要求
- **分数线库**：
  - 广东历年各批次省控线
  - 各院校各专业组投档线
  - 一分一段表（分数对应排名）

### 2.3 智能推荐模块（核心）

#### 2.3.1 推荐范围限定
为避免推荐无意义的结果，系统限定推荐范围在**考生分数 ±50 分**区间内：
- 比考生分数高太多的院校（如高出 50 分以上），往年根本考不上，推荐无意义
- 比考生分数低太多的院校（如低于 50 分），浪费分数，也不推荐
- 以考生分数为基准线，上下各扩展约 50 分（对应排名范围由一分一段表动态换算），在此范围内筛选推荐院校

> 📌 **典型场景**：一个考了 580 分的考生，系统聚焦在历年投档线约 530~630 分区间的院校。清北等顶尖院校（投档线 680+）不会被推荐，专科或过低档次的院校也不会出现。

#### 2.3.2 冲稳保分类逻辑（基于分数差值）

**核心理念**：以分数差值为主、排名差值为辅进行三档分类。分数差值更直观，考生更容易理解。

| 分类 | 标签 | 分数差值范围 | 含义 |
|------|------|-------------|------|
| 🔴 **冲** | Reach | 高于考生分数 **10~20 分** | 往年录取分略高于自己，有一定难度但值得冲刺 |
| 🟡 **稳** | Match | 与考生分数 **±10 分以内** | 往年录取分与自身接近，录取希望较大 |
| 🟢 **保** | Safety | 低于考生分数 **10~20 分** | 往年录取分明显低于自己，100% 稳妥录取 |

**分类标准细化**：
```
以考生分数 580 分为例：
  冲（Reach）：  院校投档线 590~600 分  → 跳一跳够得着
  稳（Match）：  院校投档线 570~590 分  → 正常发挥能上
  保（Safety）： 院校投档线 560~570 分  → 发挥失常也能兜底

以考生排名 18000 名为例（配合一分一段表换算）：
  冲：院校录取排名约 12000~16500 名（排名更高=更难考）
  稳：院校录取排名约 16000~20000 名（排名接近）
  保：院校录取排名约 20000~25000 名（排名更低=更好考）
```

> ⚠️ **重要说明**：分数差值和排名差值之间通过一分一段表动态换算。不同年份、不同分数段的"1分含金量"不同——高分段（如 650+）每差 1 分排名可能只差几百名，而中分段（如 500 分）每差 1 分可能就差了上千名。因此分类阈值会根据用户所在分数段动态微调。

#### 2.3.3 院校专业组信息透明化

**问题背景**：每个院校下设多个"专业组"，不同专业组的投档线差异可能很大。例如某 985 高校，热门工科专业组投档线 630 分，而冷门文科专业组可能仅 580 分。院校官网通常只展示"最低投档线"，这往往是该校最冷门专业组的分数线。

**解决方案**：
- 推荐列表中**同时展示院校和专业组**，不只看院校层面的最低分数线
- 每个专业组标注所包含的**具体专业方向**（如"专业组201：计算机类、电子信息类"），让考生一眼知道这个代码代表什么
- 对专业组进行**质量标注**：
  - 🥇 **热门专业组**：包含该校优势/热门专业，投档线较高
  - 🥈 **普通专业组**：包含常规专业，投档线中等
  - 🥉 **冷门专业组**：包含该校弱势/冷门专业，投档线较低（即"好学校的差专业"）
- 在院校详情中明确标注：该院校最高专业组投档线与最低专业组投档线的**分差**
- **风险提示**：如果考生以接近最低投档线的分数进入某院校，大概率只能进入该校最冷门的专业（组），系统需给出明确提示

#### 2.3.4 专业方向细分推荐

**场景**：考生在信息输入时选择了感兴趣的专业方向（如"工学-计算机类"），系统应当：

1. **大类 → 细分专业展开**：
   - 工学 → 计算机类 → 计算机科学与技术 / 软件工程 / 人工智能 / 网络空间安全 / 数据科学与大数据技术 / 物联网工程 ...
   - 医学 → 临床医学类 → 临床医学 / 医学影像学 / 麻醉学 / 儿科学 ...

2. **按细分专业展示投档线**：
   - 在同一院校下，展示不同细分专业（或专业组）各自的历年投档线和排名
   - 帮助考生区分"同一学校、不同专业"的录取难度差异

3. **专业方向匹配过滤**：
   - 如果考生选了"计算机类"，优先展示各院校的计算机相关专业组
   - 同时保留该院校其他专业组作为备选（降低优先级展示）

4. **专业冷热标记**：
   - 标注各专业近年热度趋势（↑上升/→平稳/↓下降）
   - 提醒考生注意"大小年"现象（某专业今年可能因去年过热而导致分数线暴涨）

#### 2.3.5 省外院校同等推荐

**原则**：作为广东省考生导向的系统，但**绝不限制院校地域范围**。

- 广东省内院校：标记 🏠 省内标签，方便考生识别
- 省外院校：正常推荐，不降权、不隐藏
- **性价比挖掘**：主动标注那些"在广东录取分数不高、但学校实力不错"的省外院校（如东北、西北、西南地区的一些好学校因地理位置偏远而在广东录取分偏低）
- **地域分析**：提供该院校所在城市的基本信息（气候、生活成本、是否一线城市等），帮助考生全面评估

**性价比标注示例**：
```
🏷️ 高性价比：XX大学（211，位于哈尔滨），在广东投档线仅 560 分，
    但该校工科实力全国排名前 50，同类院校在广东需 600+
```

#### 2.3.6 推荐输出内容（完整版）

每条推荐记录包含以下信息：
- 院校名称 + 所在地 + 层次（985/211/双一流/普通）
- 专业组代码 + **所含专业方向说明**
- 历年录取最低排名 & 分数（多年对比）
- 院校最高专业组 vs 最低专业组分差
- 录取概率估算（百分比）
- 冲/稳/保 标签
- 与考生分数的差值（+15 分 / -8 分 等直观展示）
- 选科要求匹配情况
- 专业热度趋势标记
- 省外性价比标记（如适用）

### 2.4 正式志愿填报模拟模块（冲稳保三档）

│  ├─ ...                                  │
│  🟡 稳的志愿（建议 15-20 个）              │
│  ├─ 志愿N: XX大学 - 专业组C              │
│  ├─ ...                                  │

- 志愿表保存与加载（本地存储或账号系统）

7. **招生计划数变化**：近两年是否扩招/缩招
8. **外部链接**：招生章程、院校官网、教育部学科评估结果

### 2.5 数据分析与可视化模块
- **年份难度对标图**：展示近 5 年一分一段分布曲线，标注 2026 年曲线位置，高亮难度相似年份（如 2022 与 2026 曲线接近），帮助用户直观理解"为什么今年的 600 分 ≈ 去年的 XXX 分"
- **排名趋势图**：展示目标院校近 5 年在广东的录取排名变化趋势（折线图）
- **分数-排名对照图**：展示用户分数在历年一分一段表中的位置
- **录取概率仪表盘**：以仪表盘形式展示各志愿录取概率
- **院校对比图**：多所院校横向对比（雷达图：综合实力、录取难度、就业率等）
- **专业热度分析**：各专业近年报考热度变化
- **分数线预测区间图**：展示预测的当年分数线区间及置信范围

### 2.6 辅助决策模块
- **志愿合理性检查**：
  - 是否冲稳保比例合理
  - 是否存在志愿断档（相邻志愿差距过大）
  - 志愿是否有梯度（排名从高到低排列）
  - 选科是否匹配

  - 高分低就风险评估（保的志愿是否过于保守）

## 五、推荐算法设计（详细）

### 5.0 年份难度分析与分数线预测（核心创新点）

#### 5.0.1 问题背景
每年高考试题难度不同，导致同样的分数在不同年份对应完全不同的排名。例如：
- **2022 年**：数学难度极大，600 分可能对应全省第 15000 名
- **2024 年**：整体难度适中，600 分可能只对应全省第 25000 名
- **2026 年**：数学难度再次加大（与 2022 年类似），预计分数分布与 2022 年相近

这意味着：如果只看分数，600 分在 2022 年能上的学校，在 2024 年完全上不了。**因此算法必须以排名为核心，同时利用"难度相似年份"来加权预测。**

#### 5.0.2 年份难度对标机制

```
核心思路：通过一分一段表（每个分数对应多少考生），构建"年份难度指纹"

步骤：
1. 提取近 5 年（2022-2026）每年的一分一段表
2. 选取关键锚点（如本科线、特控线、600分、550分等）对应的排名
3. 计算各年份在各锚点的排名曲线
4. 计算 2026 年与各历史年份的"难度相似度"
5. 在推荐算法中，对难度相似的年份赋予更高权重

难度相似度计算（简化版）：
  选取 N 个关键分数点（如 700, 650, 600, 550, 500, 450, 本科线, 特控线）
  对每个分数点，比较 2026 年排名与历史年份排名的偏差
  相似度 = 1 - 平均偏差率

  示例输出：
    2026 ≈ 2022（相似度 92%）  ← 难度相似年，权重最高
    2026 ≈ 2023（相似度 68%）
    2026 ≈ 2024（相似度 45%）  ← 难度差异大，权重最低
    2026 ≈ 2025（相似度 55%）
```

#### 5.0.3 加权预测模型

```
预测 2026 年某院校录取分数线：

传统做法（等权平均）：
  预测排名 = (2022排名 + 2023排名 + 2024排名 + 2025排名) / 4
  ❌ 问题：等权平均忽略了年份难度差异

改进做法（难度加权）：
  预测排名 = 2022排名 × W₂₀₂₂ + 2023排名 × W₂₀₂₃ + 2024排名 × W₂₀₂₄ + 2025排名 × W₂₀₂₅
  
  其中 W 为难度权重：
    W₂₀₂₂ = 0.35  （与2026最相似）
    W₂₀₂₃ = 0.20
    W₂₀₂₄ = 0.15  （与2026差异最大）
    W₂₀₂₅ = 0.30

  此外还需考虑：
  - 招生计划数变化（扩招/缩招）
  - 院校热度变化趋势
  - "大小年"效应（部分院校录取排名隔年波动）
```

#### 5.0.4 分数线预测输出

对每个推荐院校，系统输出：
- 📊 **预测排名区间**：基于难度加权模型的排名范围
- 📈 **预测分数区间**：将预测排名反查 2026 一分一段表，得到对应分数
- 🔍 **置信度**：基于数据离散程度给出预测置信度（高/中/低）
- 📉 **趋势箭头**：相比去年 ↑ 上升 / → 持平 / ↓ 下降

### 5.1 冲稳保分类算法（分数差值 + 排名差值双维度）

#### 5.1.1 核心原则
- **分数差值为主**：考生对"比我高 15 分"比"排名比我高 3000 名"更有直观感受
- **排名差值兜底**：排名是跨年份对比的稳定锚点，避免分数波动误导
- **一分一段表做桥梁**：将分数差值动态换算为排名差值，适配不同分数段

#### 5.1.2 推荐区间限定（±50 分）

```
推荐范围 = [考生分数 - 50, 考生分数 + 50]

以考生 580 分 / 排名 18000 为例（分数段一分≈300名）：
  推荐上限：630 分（对应排名约 8000 名）  ← 冲的最高限度
  推荐下限：530 分（对应排名约 30000 名） ← 保的最低限度

500 分考生（一分≈800名）：
  推荐上限：550 分（对应排名约 28000 名）
  推荐下限：450 分（对应排名约 68000 名）

650 分考生（高分段，一分≈150名）：
  推荐上限：700 分（对应排名约 300 名）
  推荐下限：600 分（对应排名约 7500 名）
```

> 📌 区间外的院校自动过滤：清北（投档线 680+）不会出现在 580 分考生的推荐中；专科院校也不会出现。

#### 5.1.3 三档分类标准

| 分类 | 分数差值 | 排名差异率 | 录取概率 | 说明 |
|------|---------|-----------|---------|------|
| 🔴 冲 | +10 ~ +20 分 | -8% ~ -15% | 20%~40% | 超常发挥才有机会 |
| 🟡 稳 | -10 ~ +10 分 | ±8% | 40%~80% | 正常发挥有望录取 |
| 🟢 保 | -10 ~ -20 分 | +8% ~ +20% | >80% | 兜底保障，发挥失常也能上 |

#### 5.1.4 动态阈值调整

不同分数段的"10 分含金量"不同，阈值需动态调整：

| 分数段 | 每1分≈排名数 | 冲的排名阈值 | 稳的排名阈值 | 保的排名阈值 |
|--------|------------|------------|------------|------------|
| 650+ (高分段) | ~150名 | 1200~2400名 | ±1200名 | 1200~2400名 |
| 550-650 (中分段) | ~300名 | 3000~6000名 | ±3000名 | 3000~6000名 |
| 450-550 (中低分段) | ~500-800名 | 5000~16000名 | ±6500名 | 6500~13000名 |
| <450 (低分段) | ~800-1200名 | 8000~24000名 | ±10000名 | 10000~20000名 |

### 5.2 年份权重手动调制

#### 5.2.1 设计理念
不同年份的高考难度、招生政策、报考人数都可能不同。系统默认提供两种权重模式，同时**允许用户手动调整**。

#### 5.2.2 两种默认权重模式

| 模式 | 逻辑 | 适用场景 |
|------|------|----------|
| **等权平均**（默认） | 所有年份权重相同（如 4 年各 25%） | 无明显难度差异时 |
| **难度加权**（可选） | 根据年份难度对标结果自动分配权重 | 某年份难度与今年明显相似时 |

#### 5.2.3 手动权重调制（用户可控）

在推荐页面的高级设置中，提供**年份权重滑块**：

```
年份权重调制（可手动拖拽或输入）：
┌─────────────────────────────────────────────────────┐
│  2022年 ━━━━━━━━●━━━━━━━━━━━━━  25%  (默认)         │
│  2023年 ━━━━━━━━━━━━●━━━━━━━━━  25%  (默认)         │
│  2024年 ━━━━━━━━━━━━━━━━●━━━━━  25%  (默认)         │
│  2025年 ━━━━━━━━━━━━━━━━━━━━●  25%  (默认)         │
│                                                       │
│  总计：100%                                           │
│  [重置为默认]  [应用难度对标权重]  [应用修改]          │
└─────────────────────────────────────────────────────┘
```

**使用场景**：
- 考生/家长认为某年数据更有参考价值时，可手动调高该年权重
- 系统提供"一键难度对标"：点击后自动按 5.0 节的难度相似度算法填入权重
- 不调整时默认等权平均，保证最简单使用体验

#### 5.2.4 权重应用的推荐计算

```
预测排名 = Σ(每年录取排名 × 该年权重)

其中权重自动归一化，确保总和 = 100%
用户调整后实时重新计算所有推荐结果
```

### 5.3 录取概率估算模型

```
录取概率 = f(排名差异, 趋势因素, 计划数变化, 选科热度, 年份权重)

MVP 实现：
- 基于近 4 年排名差异的加权平均（权重默认 25% 各年，用户可调）
- 输出录取概率等级：
  🟢 极高  >85%    （保底院校，历年排名远低于考生）
  🟡 较高  65~85%   （稳的院校，历年排名接近或略低）
  🟠 中等  40~65%   （偏冲的院校，有一定希望）
  🔴 较低  20~40%   （冲的院校，需要超常发挥）
  ⚫ 极低  <20%     （几乎不可能，仅作参考）
```

### 5.4 录取概率细化因素（后续迭代）

在 MVP 阶段使用排名差异的加权平均，后续可引入：
- 招生计划数变化：扩招 → 录取概率上调；缩招 → 下调
- 院校热度趋势：近年热度上升 → 分数线可能上涨 → 下调
- "大小年"修正：检测院校是否存在隔年波动规律
- 选科竞争度：物理/历史类人数比例变化影响竞争激烈程度

### 5.3 选科匹配校验
```
规则引擎：
  考生选科 = {物理, 化学, 生物}
  专业组要求 = "物理,化学"    → ✅ 匹配
  专业组要求 = "物理,政治"    → ❌ 不匹配
  专业组要求 = "不限"         → ✅ 匹配
```
