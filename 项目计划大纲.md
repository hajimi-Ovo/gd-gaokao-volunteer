# 广东省高考志愿填报参考系统 — 项目计划大纲

> **目标用户**：广东省高考考生（如你的弟弟）
> **核心价值**：根据高考分数与排名，结合历年数据，提供科学合理的志愿填报参考建议

---

## 一、项目概述

### 1.1 项目定位
面向广东省高考生的志愿填报辅助参考平台，帮助考生在"新高考"模式下（3+1+2），根据自身分数、排名、选科组合，智能推荐院校与专业，并按照"冲、稳、保"三级策略生成志愿填报方案。

### 1.2 广东省高考特点（需重点考虑）
- **模式**：新高考"3+1+2"（语数英 + 物理/历史 + 两门选考）
- **批次划分**：本科批、专科批（本科批合并，不再分一本二本）
- **分数线类型**：
  - 特殊类型招生控制线（相当于原一本线）
  - 本科线
  - 专科线
- **选科要求**：各院校专业对选考科目有明确要求（物理/历史/不限等）
- **排名的重要性**：广东考生数量大，排名比分数更具参考价值
- **志愿数量**：本科批可填 45 个院校专业组志愿

### 1.3 核心原则
- **排名优于分数**：每年试题难度不同，分数线波动大，但排名相对稳定。例如 2026 年数学难度较大（与 2022 年类似），同样的分数对应的排名会比"简单年份"更高，因此排名才是跨年份对比的"硬通货"
- **年份难度对标**：通过一分一段表分析各年份的分数-排名分布曲线，识别出"难度相似年份"（如 2026≈2022），在预测当年分数线时，给难度相似年份的权重更高，从而提高预测准确度
- **冲稳保策略**：科学划分三个层次，合理分配志愿数量
- **数据驱动**：基于近 3-5 年官方录取数据进行推算，难度年份加权
- **选科匹配**：严格校验考生选科与院校专业要求的匹配性

## 二、功能模块设计

### 2.1 用户输入模块（个人信息采集）
| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 高考总分 | 考生查询到的最终分数 | ✅ 必填 |
| 全省排名（位次） | 广东省教育考试院公布的排名 | ✅ 必填 |
| 首选科目 | 物理 / 历史 | ✅ 必填 |
| 再选科目（2门） | 化学/生物/政治/地理 中任选2门 | ✅ 必填 |
| 目标城市偏好 | 省内/省外/无偏好/具体城市 | 可选 |
| 目标专业方向 | 工学/医学/经管/文史/师范/艺术等 | 可选 |
| 院校层次偏好 | 985/211/双一流/公办本科/民办本科/专科 | 可选 |
| 学费承受范围 | 公办标准/可接受民办/中外合作办学 | 可选 |

**扩展考虑**：
- 是否有加分项（少数民族、华侨等）
- 是否有体检限制（色盲色弱等影响专业选择）
- 是否为农村专项/地方专项计划考生

### 2.2 数据查询与浏览模块
- **院校库**：
  - 全国高校基本信息（所在地、层次、类型、是否985/211/双一流）
  - 各院校在广东的招生计划（分专业组、分专业）
  - 院校历年最低投档线 & 对应排名
  - 院校各专业组最低投档线 & 对应排名
- **专业库**：
  - 各专业详细介绍、培养目标、主要课程
  - 就业方向与前景分析
  - 各专业在广东的历年录取分数 & 排名
  - 专业对选科的要求
- **分数线库**：
  - 广东历年各批次省控线
  - 各院校各专业组投档线
  - 一分一段表（分数对应排名）

### 2.3 智能推荐模块（核心）
**推荐算法逻辑**：
1. **排名换算**：将用户排名与历年各院校录取排名进行比对
2. **差值计算**：计算用户排名与院校历年录取排名的差值
3. **冲稳保分类**：
   - 🔴 **冲（Reach）**：院校录取排名高于用户排名 0~15%（即用户排名略低于往年录取排名）
   - 🟡 **稳（Match）**：院校录取排名在用户排名 ±5% 范围内
   - 🟢 **保（Safety）**：院校录取排名低于用户排名 5~20%（即用户排名明显高于往年录取排名）
4. **选科过滤**：排除不符合选科要求的院校专业组
5. **排序输出**：按推荐优先级排序

**推荐输出内容**：
- 推荐院校名称 + 专业组
- 历年录取最低排名 & 分数
- 录取概率估算（百分比）
- 冲/稳/保 标签
- 与用户排名的差距
- 选科要求匹配情况
- 院校所在地 & 层次

### 2.4 正式志愿填报模拟模块（冲稳保三档）
模拟广东省教育考试院志愿填报界面，按三个档位组织：

```
┌─────────────────────────────────────────┐
│  🔴 冲的志愿（建议 15-20 个）              │
│  ├─ 志愿1: XX大学 - 专业组A              │
│  ├─ 志愿2: XX大学 - 专业组B              │
│  ├─ ...                                  │
│  🟡 稳的志愿（建议 15-20 个）              │
│  ├─ 志愿N: XX大学 - 专业组C              │
│  ├─ ...                                  │
│  🟢 保的志愿（建议 5-10 个）              │
│  ├─ 志愿M: XX大学 - 专业组D              │
│  └─ ...                                  │
└─────────────────────────────────────────┘
```

**功能特性**：
- 用户可从推荐列表中勾选加入志愿表
- 支持拖拽排序调整志愿顺序
- 实时显示已选志愿数量 / 各档位数量
- 志愿表导出（PDF / 图片 / 打印）
- 志愿表保存与加载（本地存储或账号系统）

### 2.5 数据分析与可视化模块
- **年份难度对标图**：展示近 5 年一分一段分布曲线，标注 2026 年曲线位置，高亮难度相似年份（如 2022 与 2026 曲线接近），帮助用户直观理解"为什么今年的 600 分 ≈ 去年的 XXX 分"
- **排名趋势图**：展示目标院校近 5 年在广东的录取排名变化趋势（折线图）
- **分数-排名对照图**：展示用户分数在历年一分一段表中的位置
- **录取概率仪表盘**：以仪表盘形式展示各志愿录取概率
- **院校对比图**：多所院校横向对比（雷达图：综合实力、录取难度、就业率等）
- **专业热度分析**：各专业近年报考热度变化
- **分数线预测区间图**：展示预测的当年分数线区间及置信范围

### 2.6 辅助决策模块
- **志愿合理性检查**：
  - 是否冲稳保比例合理
  - 是否存在志愿断档（相邻志愿差距过大）
  - 志愿是否有梯度（排名从高到低排列）
  - 选科是否匹配
- **风险评估**：
  - 滑档风险评估（冲的志愿是否过多）
  - 高分低就风险评估（保的志愿是否过于保守）

## 五、推荐算法设计（详细）

### 5.0 年份难度分析与分数线预测（核心创新点）

#### 5.0.1 问题背景
每年高考试题难度不同，导致同样的分数在不同年份对应完全不同的排名。例如：
- **2022 年**：数学难度极大，600 分可能对应全省第 15000 名
- **2024 年**：整体难度适中，600 分可能只对应全省第 25000 名
- **2026 年**：数学难度再次加大（与 2022 年类似），预计分数分布与 2022 年相近

这意味着：如果只看分数，600 分在 2022 年能上的学校，在 2024 年完全上不了。**因此算法必须以排名为核心，同时利用"难度相似年份"来加权预测。**

#### 5.0.2 年份难度对标机制

```
核心思路：通过一分一段表（每个分数对应多少考生），构建"年份难度指纹"

步骤：
1. 提取近 5 年（2022-2026）每年的一分一段表
2. 选取关键锚点（如本科线、特控线、600分、550分等）对应的排名
3. 计算各年份在各锚点的排名曲线
4. 计算 2026 年与各历史年份的"难度相似度"
5. 在推荐算法中，对难度相似的年份赋予更高权重

难度相似度计算（简化版）：
  选取 N 个关键分数点（如 700, 650, 600, 550, 500, 450, 本科线, 特控线）
  对每个分数点，比较 2026 年排名与历史年份排名的偏差
  相似度 = 1 - 平均偏差率

  示例输出：
    2026 ≈ 2022（相似度 92%）  ← 难度相似年，权重最高
    2026 ≈ 2023（相似度 68%）
    2026 ≈ 2024（相似度 45%）  ← 难度差异大，权重最低
    2026 ≈ 2025（相似度 55%）
```

#### 5.0.3 加权预测模型

```
预测 2026 年某院校录取分数线：

传统做法（等权平均）：
  预测排名 = (2022排名 + 2023排名 + 2024排名 + 2025排名) / 4
  ❌ 问题：等权平均忽略了年份难度差异

改进做法（难度加权）：
  预测排名 = 2022排名 × W₂₀₂₂ + 2023排名 × W₂₀₂₃ + 2024排名 × W₂₀₂₄ + 2025排名 × W₂₀₂₅
  
  其中 W 为难度权重：
    W₂₀₂₂ = 0.35  （与2026最相似）
    W₂₀₂₃ = 0.20
    W₂₀₂₄ = 0.15  （与2026差异最大）
    W₂₀₂₅ = 0.30

  此外还需考虑：
  - 招生计划数变化（扩招/缩招）
  - 院校热度变化趋势
  - "大小年"效应（部分院校录取排名隔年波动）
```

#### 5.0.4 分数线预测输出

对每个推荐院校，系统输出：
- 📊 **预测排名区间**：基于难度加权模型的排名范围
- 📈 **预测分数区间**：将预测排名反查 2026 一分一段表，得到对应分数
- 🔍 **置信度**：基于数据离散程度给出预测置信度（高/中/低）
- 📉 **趋势箭头**：相比去年 ↑ 上升 / → 持平 / ↓ 下降

### 5.1 排名差异率计算
```
差异率 = (院校最低录取排名 - 考生排名) / 考生排名 × 100%

分类标准：
  差异率 < -15%           → 🔴 冲（难度较大）
  -15% ≤ 差异率 ≤ -5%     → 🔴 冲（有一定希望）
  -5% < 差异率 < 0%       → 🟡 稳（希望较大）
  0% ≤ 差异率 ≤ 10%       → 🟢 保（把握很大）
  差异率 > 10%            → 🟢 保（完全稳妥）
```

### 5.2 录取概率估算模型
```
录取概率 = f(排名差异, 趋势因素, 计划数变化, 选科热度)

简化版（MVP）：
- 基于近3年排名差异的加权平均
- 权重：最近年份更高（如 2024=50%, 2023=30%, 2022=20%）
- 输出：大致录取概率（高>80% / 中50-80% / 低20-50% / 极低<20%）
```

### 5.3 选科匹配校验
```
规则引擎：
  考生选科 = {物理, 化学, 生物}
  专业组要求 = "物理,化学"    → ✅ 匹配
  专业组要求 = "物理,政治"    → ❌ 不匹配
  专业组要求 = "不限"         → ✅ 匹配
```

## 七、待决策事项

### 7.1 需确认的技术选型
| 决策项 | 选项 | 建议 |
|--------|------|------|
| 前端框架 | React vs Vue | React（生态更好） |
| UI 框架 | Ant Design vs TDesign | Ant Design（表格组件强） |
| 是否全栈 | 纯前端 vs 前后端分离 | MVP 纯前端（快速验证） |
| 数据规模 | 只做本科批 vs 含专科 | 优先本科批 |
| 用户系统 | 需要 vs 不需要 | MVP 不需要 |
| 部署平台 | Vercel vs Cloudflare Pages | Vercel（最简）/ Cloudflare Pages（国内快） |
| 代码托管 | GitHub vs Gitee | GitHub（与 Vercel 联动最方便） |

### 7.2 需确认的功能边界
- 是否包含专科批次院校？
- 是否包含提前批（军校、警校、艺术体育类）？
- 是否包含中外合作办学/独立学院？
- 是否需要用户注册登录功能？
- 是否考虑"地方专项计划"、"少数民族预科"等特殊类型？
- 是否需要多用户协作（如家长与考生共享志愿表）？

### 7.3 数据风险与免责
- 所有推荐仅供参考，最终决定权在用户
- 需在页面显著位置添加免责声明
- 数据来源尽量标注，确保可追溯
